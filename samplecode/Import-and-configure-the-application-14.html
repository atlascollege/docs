Import and configure the application

Import and configure the application

工程准备

  1. 参考Obtain the Source Code下载videoanalysis源码包，目录包含如下子目录，其中子目录videoanalysis为工程源码，MyModel子目录为模型文件夹。

    videoanalysis
├─videoanalysisapp
└─MyModel

  2. 将Obtain the Source Code中的模型包下载后放入videoanalysis源码包的MyModel子目录中。

    videoanalysis
├─videoanalysisapp
└─MyModel
   └─object_detection_model
   └─car_color_model
   └─car_type_model
   └─pedestrian_attr_model

  3. 在videoanalysis源码包下，选中videoanalysisapp、MyModel两个子目录，压缩为videoanalysis.zip包。

工程导入

  1. 在Mind Studio中选择“File“菜单，单击“Upload Project“菜单项，如图1 上传工程所示；

    图 1 上传工程

  2. 选择工程准备环节制作的“videoanalysis.zip“导入，导入成功，单击Close按钮，如图2所示。

    图 2 上传成功

工程配置

  1. 在左侧“Projects Explorer“导航栏中选“择videoanalysis“工程，双击“videoanalysis.mind“文件，在画布区域出现如图3所示流程图。

    图 3 流程编排

  2. 单击“video_decode“图标，在右侧“Property“菜单配置输入视频源路径，如图4所示，当前支持配置两个通道视频输入，可以配置为保存在Atlas DK开发板本地中的mp4文件或者RTSP视频源地址。

    图 4 配置视频源属性

    说明：
    因为Atlas DK开发板SD卡空间有限，如果视频文件过大，建议通过磁盘挂载的方式读取。

  3. 单击“object_detection_post“图标，在右侧Property菜单“Advance > Configs“下配置目标检测置信度“Confidence“的值，大小范围为0-1的浮点数。数值越大表示置信度越高。

    图 5 设置目标检测置信度属性

  4. 单击“video_analysis_post“图标。

    • 在右侧Property菜单“Advance > Configs“中配置“presenter_server_ip“（部署的presenter agent的服务端IP地址）和“presenter_server_port“，即3中“config.conf“文件的“presenter_server_ip“和“presenter_server_port“的值。

    • 自定义配置app_name，app_name的值为输出展示页面中的App Name。

      图 6 后处理参数配置
